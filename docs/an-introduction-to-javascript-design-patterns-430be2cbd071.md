设计模式简介

![](img/99c558d78809bce1efbf70786e9da727.png)

Photo by [Derek Oyen](https://unsplash.com/@goosegrease?utm_source=medium&utm_medium=referral) on [Unsplash](https://unsplash.com?utm_source=medium&utm_medium=referral)

如果你想成为一名成功的 JavaScript 开发者，就要编写清晰、健康、易于维护的代码。你克服了有趣的问题，这些问题虽然独特，但不一定需要独特的解决方案。您可能已经发现自己开发的代码类似于您以前解决的一个完全不相关的问题的答案。虽然您可能没有意识到，但是您已经使用了 JavaScript 设计模式。软件设计中常见问题的可重用解决方案被称为设计模式。

在该语言存在的过程中，来自任何语言社区的大量开发人员都创建并测试了许多这样的可重用解决方案。这种解决方案非常有用，因为它们使我们能够以最佳方式编写代码，同时仍然解决手头的问题，这要归功于许多人的共同经验。

针对经常出现的软件问题的高级面向对象解决方案被称为设计模式。可重用的设计和事物的交互是模式的核心。当讨论复杂的设计解决方案时，每个模式都有一个名称，并被添加到词典中。

考虑到它们为软件设计中经常出现的问题提供了典型的答案，设计模式是软件开发的基本组成部分。设计模式本质上是可以用来以最佳方式管理重复出现的主题的思想，而不是提供特定的软件组件。

在过去的几年中，web 开发生态系统发生了显著的变化。一些众所周知的设计模式可能不再像以前那样有用了，但其他一些已经发展到使用尖端技术来解决当代问题。

以下是设计模式的主要优势:

*   **它们是屡试不爽的解决方案:**由于许多开发人员经常使用设计模式，你可以确信它们是有效的。此外，你可以肯定他们经历了几次修订，优化可能是适用的。
*   **它们可以很容易地被重用:**由于它们独立于单个问题，设计模式描述了一个可重用的解决方案，可以通过修改来解决许多单独的问题。
*   **他们表达自己:**大型解决方案可以用设计模式来优雅地解释。
*   他们促进交流:当开发人员熟悉设计模式时，他们可以更容易地讨论特定问题的替代解决方案。
*   **它们避免了代码重构的需要:**如果一个应用程序是用设计模式创建的，那么通常情况下，你在未来不需要这样做，因为使用正确的设计模式来解决特定的问题已经是最好的行动方案了。
*   **它们减少了代码库的大小:**设计模式通常比其他解决方案需要更少的代码，因为它们是优雅和理想的解决方案。

# 什么是设计模式？

正如我前面提到的，设计模式是软件设计中经常出现的问题的可重复的解决方案。让我们看看一些设计模式类别。

## 原型模式

一个图案怎么做出来？想象一下，如果你发现了一个普遍存在的问题，并想出了你自己独特的解决方法，但却没有被广泛认可或记录。每当您遇到这个问题时，您都会利用这个解决方案，并且您相信它对开发人员社区是可重用的和有用的。

是不是马上就开始重复了？感谢上帝，没有。即使使用适当的编码方法，也有可能错误地识别出实际上不符合任何一种模式的事物中的模式。

什么告诉你你认为你认识的任何东西实际上是一个设计模式？

通过向其他开发人员寻求反馈，学习如何从零开始创建一个模式，并精通现有的模式。原型模式是一个模式在被认为是完全成熟的模式之前必须经历的阶段。

如果一个原型模式成功地完成了许多开发人员和场景的一段时间的测试，其中该模式展示了它的效用并产生了期望的结果，那么它就被认为是一个未来的模式。要制作一个被社区认可的成熟模式，需要做大量的工作和文档——其中大部分超出了本文的范围。

## 反模式

反模式与设计模式相反，代表优秀的实践。

修改`Object`类原型将是反模式的一个例子。考虑这样一种情况，您更改了这个原型，因为几乎所有的 JavaScript 对象都继承自`Object`(记住 JavaScript 采用基于原型的继承)。从`Object`原型继承的所有对象，包括大多数 JavaScript 对象，都会受到对象原型变化的影响。这种情况是一场正在形成的灾难。

另一个类似于前面讨论的例子是改变你不拥有的东西。从在应用程序的许多实例中使用的对象中重写函数就是一个例子。想象一下，如果你和一个大团队一起工作，这会带来多大的困难；您会很快陷入命名冲突、不兼容的实现和维护问题。

就像了解所有有效的策略和实践是有益的一样，了解任何无效的策略和实践也是至关重要的。通过这种方式，您能够识别它们并提前避开错误。

# 设计模式分类

有几种方法可以对设计模式进行分类，但是，下面是最常用的一种方法:

*   **创造性的**设计模式
*   **结构**设计模式
*   **行为**设计模式
*   **并发**设计模式
*   **建筑**设计模式
