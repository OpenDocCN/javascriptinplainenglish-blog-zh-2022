如何创建记录自身的文档

为了简化苗条组件文档的创建，我创建了2个NPM包。第一个，@ El 3um 4s/svelte-get-component-info，任务是解析组件并提取一个包含基本信息的JSON对象。第二个，@ El 3um 4s/svelte-component-info，负责将这些信息转换成屏幕上可读的格式。

为什么我要把项目一分为二？因为通过这种方式，如果有人想这样做，就有可能创建不同的图形展示。

npm i -D @el3um4s/svelte-get-component-info
