包装

@mantine/hooks–用于状态和UI管理的40多个钩子的集合

@mantine/styles–基于情感的css-in-js库，用于所有Mantine组件

@mantine/core–核心组件库–100多个组件，从@mantine/styles导出所有组件

@mantine/form–表单管理库

@mantine/notifications–功能全面的通知系统

@mantine/spotlight–Ctrl + K您应用的命令中心

@mantine/prism–使用 prism-react-renderer 构建的代码高亮显示

@mantine/rte–基于Quill的富文本编辑器

@mantine/dropzone–管理拖放到某个区域或整个屏幕的文件

@mantine/carousel–转盘组件

@mantine/nprogress–导航进度

@mantine/modals–集中模式管理器

@mantine/ssr–服务器端渲染实用程序

@mantine/next–next . js的服务器端渲染实用程序

@mantine/remix–用于混音的服务器端渲染工具

gatsby-plugin-mantine–用于设置服务器端渲染的Gatsby插件

eslint-config-mantine–ESLint和更漂亮的配置，用于所有Mantine项目

文档

如何入门？

npm install @mantine/core @mantine/hooks @emotion/react

用mantine provider包装您的应用程序根组件:

import { MantineProvider } from '@mantine/core';function Demo() {return (<MantineProvider withGlobalStyles withNormalizeCSS><App /></MantineProvider>);}

一切就绪！现在，您可以在应用程序中的任何地方导入和使用Mantine组件:

import { Button } from '@mantine/core';function Demo() {return <Button>Click me!</Button>;}

对于框架用户来说，应用起来会更容易。如创建react app :

首先，生成新的应用程序，

npx create-react-app my-mantine-app — template typescript

然后安装Mantine，

npm install @mantine/core @mantine/hooks @emotion/react

将App.tsx的内容替换为:

import { MantineProvider, Text } from '@mantine/core';export default function App() {return (<MantineProvider withGlobalStyles withNormalizeCSS><Text>Welcome to Mantine!</Text></MantineProvider>);}

一切就绪！启动开发服务器:

npm run dev

UI最棒的部分是它内置了123个响应组件。

你认为哪个更好？Mantine还是Material UI？我们来对比一下。

1.Mantine是一个具有原生黑暗主题支持的React组件库(由 mantinedev )。而MUI Core是React UI库的集合，用于更快地发布新特性。从我们的全加载组件库Material UI开始，或者将您自己的设计系统引入我们的生产就绪组件。(由梅伊)。

2.Mantine是用黑暗主题模式的TypeScript构建的。材质UI是用JavaScript搭建的。
