用GraphQL包装一个RESTful API并部署到Vercel

介绍

作为一名web开发人员，我敢打赌您已经非常熟悉RESTful APIs，并且已经用它们构建了非常棒的应用程序。RESTful API被视为设计API的常规标准，它使用HTTP方法来执行CRUD操作。GET检索，POST创建，PUT更新，DELETE删除。然而，RESTful API也有一些限制。例如，过度提取意味着有效负载返回的数据比客户端需要的多。有时，客户端需要发出多个请求来获取必要的数据。

RESTful vs GraphQL

为了克服这些问题，脸书发布了GraphQL，这是一种请求数据并让客户端精确指定应该返回什么数据的语法。GraphQL还允许将多个模式组合成一个模式，拼接模式有助于客户端通过一个请求检索所有数据。

本文将介绍如何使用Next.js API路由构建GraphQL API服务。我们将用GraphQL API包装我最喜欢的RESTful API之一randomuser.me。

这里，问候解析器函数返回一个静态的 Hello World！ 消息。问候查询非常简单，因为这个查询不需要参数。

用GraphQL包装randomuser.me

是时候用GraphQL包装 randomuser.me RESTful应用了。

randomuser.me是一个RESTful API，可以生成随机用户数据。查看官方文件了解更多详情。

首先，根据用户负载在typeDefs中创建一个模式。

random user . me API接受参数来指定结果，比如要返回的用户数量和返回用户的性别。因此，我们应该在用户查询中添加参数。

一些众所周知的属性类型可以用graphql-scalars工具进行验证。例如，【UUID】和 电子邮件地址 标量有助于使用正则表达式验证字符串数据。

# yarn add graphql-scalars graphql-parse-resolve-info

然后，为用户查询创建解析程序。

当GraphQL服务接收到用户查询时，用户解析器将从查询中提取所需的字段，并构建inc参数。inc参数是指定 randomuser.me 结果包含的字段。然后，来自用户查询的所有参数将被传递到查询字符串中的 randomuser.me 。

由于graphql-scalars用于验证，我们也应该在标量解析器中合并它们。

最后，构建用户查询，我们可以用GraphiQL工具测试它。
