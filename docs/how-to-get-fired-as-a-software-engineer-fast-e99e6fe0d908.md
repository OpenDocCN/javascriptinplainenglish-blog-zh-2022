# 软件工程师如何被炒，快。

> 原文：<https://javascript.plainenglish.io/how-to-get-fired-as-a-software-engineer-fast-e99e6fe0d908?source=collection_archive---------2----------------------->

![](img/5f440dd4e53adcad4b075b8be36336ef.png)

Photo by [Antony Shkraba](https://www.pexels.com/@anthonyshkraba-production) on [Pexels](https://pexels.com).

我以自由职业者的身份为一家中型公司工作，我们必须通过雇佣新的软件工程师来扩大我们的团队。但不是任何软件工程师。

有趣的是，不是一个工程师，不是两个，而是三分之二的被雇佣者，在试用期的三个月内被解雇。**为什么会这样？** —你可能会问。

为了解开这个谜，我必须告诉你我们提供的职位的要求以及招聘过程的一些额外背景。

我们正在寻找高级全栈工程师，这是很难找到这些天。

这家公司的招聘人员拼命寻找有能力的人。他们在多个地方发布招聘信息，甚至在 [*有问必答*](https://craigslist.org/) *上。*

我们收到了多份申请，大多数都不令人满意。几个月后，我们得到了一些有趣的候选人(在纸上)，我们决定对他们进行面试。

我负责面试这些候选人。我就不告诉你这个过程的细节了。但在 3/3 的情况下，我给招聘人员的反馈大多是负面的。尽管如此，一些候选人声称曾在大公司工作过，如 IBM、SpaceX 和 SpaceX。

我拒绝这些候选人的主要原因是他们不符合高级**全栈工程师职位的要求。**

如果他们几乎不了解前端开发，或者如果他们不能解决编码挑战，他们怎么能满足这些需求呢？

更糟糕的是，这些候选人对公司一无所知，他们加入我们的唯一动机就是薪水和工作条件(我必须承认，这非常好)。

由于我拒绝了所有的候选人，招聘团队决定无视我的意见，无论如何都要聘用他们。

一开始，我不明白他们为什么要雇佣他们。但我现在有点明白了。我们有最后期限要满足，我们只需要继续前进，交付代码。

*   **猜猜发生了什么？**
*   他们被解雇了，所有人。
*   **但是为什么呢？**

第一个候选人对前端和后端开发一无所知。他主要在 Swift 中编写代码，构建 iOS 应用程序。当他过渡到 JavaScript 和 React 时，他迷失了。他不知道 React 是什么，也不知道响应式设计，对 HTML 和 CSS 的了解也非常有限。他不断地交付除了错误、重复代码和混乱之外的任何东西。

第二个候选人自称是大四学生，被聘为大四学生，但表现得像大三学生。你可以看到并感觉到他在这个职位上的无能。他正在努力交付高质量的软件，更糟糕的是，他没有满足最后期限，并且不断地推迟项目。他在三个月内被解雇了。

最后一个候选人是个幸运儿。当他提出申请时，公司更加不顾一切地雇佣某人，不管他们的技能如何，同时给他们“高级”头衔和相应的薪水。他们应该在 TypeScript 中建立一个简单的后端基础设施，这将是一个大规模应用程序的基础。他不知道如何做这件事。这对于一个**高级**全栈工程师来说是不能接受的。我希望你同意…他加入的同一个月就被解雇了。

# 结论

在你的编程生涯中追求更高的水平是好的，但是你永远不应该在你的经验或技能上撒谎，因为这可能会给你带来更多的伤害。没有人希望被解雇或被贴上这样的标签。

如果你足够幸运，即使你还不具备所需的技能和/或态度，一些公司也会聘用你为高管。但是如果你认为你有，确保你继续**尽可能多地学习**，**按时交付**和**做出积极的改变**。

> ⚠️成为一名高级软件工程师并不意味着你到达了山顶。

有好的和坏的高级开发人员，明智地选择你想雇佣的和/或你想成为的。

# **补充思路:**

我想补充一下结论，对于一个人来说，在前端和后端(即:全栈)都成为高级是极其困难的。我相信你可以精通两者，但你必须在其中一个或另一个方面更强/更熟练，而不是同时精通两者。我认为编程行业需要重新评估“高级全栈”的真正含义。
