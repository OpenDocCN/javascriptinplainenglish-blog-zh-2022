通过示例了解源地图。

嗨伙计们！

Source maps无处不在！每次你打开一个网站，它就会加载成吨的缩小的JavaScript包，其中一些有source maps。它极大地简化了调试，所以经验法则是将它们添加到开发构建中。

使用特殊注释将源映射嵌入到生成的源中。这些注释可能包含使用数据URI 的整个源地图，或者可能引用外部URL或文件。

node . js中的源地图

在我们的例子中，使用了数据URL。下面是源图的一个例子:

{
    version : 3,
    file: "out.js",
    sourceRoot : "",
    sources: ["foo.js", "bar.js"],
    names: ["src", "maps", "are", "fun"],
    mappings: "AAgBC,SAAQ,CAAEA"
}

受益于源图的工具之一是:mock-import。它有助于mock导入你正在使用的，所以你可以很容易地测试📼Supertape,有史以来最伟大的跑步测试者😏！).

🗺源地图

当你需要一个sourcemap时，只需打电话就能轻松拥有🐊输出用:

  • ✅ sourceFileName
  • ✅ sourceMapName
putout(source, {
    fix: false,
    sourceFileName: 'hello.js',
    sourceMapName: 'world.js',
    plugins: [
        'remove-unused-variables',
    ],
});
// returns
({
    code: '\n' +
    `    const hello = 'world';\n` +
    `    const hi = 'there';\n` +
    '    \n' +
    '    console.log(hello);\n' +
    '// {"version": 3, ...}',
    places: [{
        rule: 'remove-unused-variables',
        message: '"hi" is defined but never used',
        position: {line: 3, column: 10},
    }],
});

今天就到这里吧！有一张漂亮的地图🦉 🥳!
